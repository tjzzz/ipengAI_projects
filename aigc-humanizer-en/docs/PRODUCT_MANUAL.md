# Huma 产品使用手册

> **适用对象**：用户、客服、运营  
> **适用版本**：激活码配额包版  
> **最后更新**：2026-07-15

## 1. 产品概述

Huma 是一款英文 AI 写作痕迹检测与自然化改写工具，当前更适合处理：

- 英文报告
- 英文长文稿
- 英文分析文档
- 英文项目材料
- 段落较长、结构完整的英文文稿

当前不主打：

- 中文内容
- 短广告文案
- 商品标题
- 纯翻译
- 代写服务

## 2. 用户主流程

```text
打开 Huma → 上传/粘贴英文文本 → 检测 AI 率
→ 查看检测结果和修改建议
→ 点击改写
→ 免费额度内：直接改写
→ 超出免费额度：优先使用词数余额
→ 余额不足：进入支付流程
```

检测结果对匿名用户开放；改写、兑换激活码、查看订单需要登录。

## 3. 检测功能

支持输入：

| 输入方式 | 说明 |
|---|---|
| 粘贴文本 | 直接粘贴英文文本 |
| 上传 `.docx` | Word 文档 |
| 上传 `.pdf` | PDF 文档 |
| 上传 `.txt` | 纯文本 |
| 上传 `.md` | Markdown |

检测结果包括：

- AI 率总分
- 风险等级
- 多维子评分
- 修改建议
- 词数与价格预估

## 4. 改写与计费

### 4.1 免费改写

如果文本词数小于等于 `FREE_WORD_LIMIT`，登录后可免费改写。

当前线上配置：

| 配置项 | 当前值 | 说明 |
|---|---:|---|
| `FREE_WORD_LIMIT` | 100 词 | 免费改写词数上限 |
| `FREE_DAILY_REWRITES` | 2 次/天 | 每用户每日免费改写次数 |
| `PRICE_PER_1000_WORDS` | ¥14.9 | 原站内按词计价 |

### 4.2 激活码配额包

闲鱼/淘宝售卖的是“兑换码配额包”。

用户购买后，卖家发送一个 Huma 激活码。用户登录 Huma，在页面右上角点击「兑换码」，输入激活码后，账户会增加对应词数余额。

当前销售 SKU：

| SKU | 词数 | 售价 |
|---|---:|---:|
| 尝鲜版 | 1000 词 | ¥12.9 |
| Basic 版 | 2000 词 | ¥25 |

### 4.3 余额改写

当文本超过免费额度时，系统会先检查用户账户词数余额：

```text
余额 >= 本次文本词数 → 扣减余额并改写
余额 < 本次文本词数 → 返回余额不足，引导支付
```

用户体验：

- 余额足够：按钮显示「使用余额改写（xx词）」。
- 余额不足但大于 0：按钮显示「余额不足，付费改写 ¥xx」。
- 改写成功：导航栏余额同步更新。
- 改写失败：系统自动退回本次已扣词数。

### 4.4 支付兜底

当余额不足时，用户仍可走原支付流程：

```text
点击改写 → /api/rewrite 返回 402 need_payment
→ 前端打开支付弹窗
→ 创建支付订单
→ 支付宝/Mock 支付
→ 支付成功后后台改写
```

如果用户余额足够，前端不会预创建支付订单，避免产生无用 pending 订单。

## 5. 激活码兑换

### 5.1 用户操作

1. 登录 Huma。
2. 点击导航栏「兑换码」。
3. 输入激活码，例如 `HUMA-XXXX-XXXX-XXXX`。
4. 点击「兑换」。
5. 成功后显示当前词数余额。

### 5.2 常见错误

| 错误 | 含义 |
|---|---|
| 请输入激活码 | 用户未输入 |
| 激活码格式不正确 | 字符过短或格式异常 |
| 激活码不存在 | 后台没有这个码 |
| 该激活码已被使用 | 码已经兑换过，不能重复使用 |

### 5.3 安全规则

- 激活码只能兑换一次。
- 兑换码状态从 `unused` 改为 `redeemed`。
- 兑换码状态更新和用户余额增加在同一事务中完成。
- 并发请求下，只有第一个兑换请求会成功。

## 6. 订单与下载

改写完成后会生成订单记录：

| `payment_status` | 含义 |
|---|---|
| `free` | 免费额度内改写 |
| `balance` | 使用词数余额改写 |
| `pending` | 支付订单等待付款 |
| `paid` / `processing` / `completed` | 支付成功后的后台改写状态 |

订单结果可复制或下载。付费订单 7 天内可再次改写，具体以 `orders` 相关接口逻辑为准。

## 7. 后台生成激活码

管理员进入后台后，切换到「激活码」Tab：

1. 输入生成数量。
2. 输入每码词数。
3. 点击「生成激活码」。
4. 复制未使用的激活码，通过闲鱼/淘宝私信发给用户。

后台可查看：

- 总码数
- 已使用数量
- 未使用数量
- 已兑换总词数
- 每个码的兑换用户和兑换时间

## 8. 客服话术

### 用户问：怎么使用？

```text
拍下后我会发你 Huma 兑换码。
你登录 writer.ipengai.cn，点击右上角「兑换码」，输入后就会获得对应词数额度。
之后上传或粘贴英文文本，点击改写时会优先扣这个额度。
```

### 用户问：一定能降到多少吗？

```text
AI 率结果会受原文质量、文本长度和检测工具影响。
页面展示的是效果示例，不承诺固定数值。
Huma 主要做英文表达自然化和机器感优化。
```

### 用户问：能不能代写？

```text
不提供代写服务。
Huma 只做已有英文内容的表达优化和自然化改写。
```

## 9. 废弃说明

以下旧逻辑不再作为当前产品说明：

- “超过免费额度直接支付”已经废弃，现在优先使用词数余额。
- “免费额度统一 200/500 词”已经废弃，以 `config.py` 的 `FREE_WORD_LIMIT` 为准，当前为 100 词。
- “会员/月卡/订阅制”暂不作为当前上架模型；当前销售模型是兑换码配额包。
- “跨境电商/外贸文案”为可扩展场景，不是当前主定位；当前主定位是英文报告和英文长文稿。
